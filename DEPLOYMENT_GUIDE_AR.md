# دليل نشر موقع CurlyEllie على Railway

## مقدمة

هذا الدليل يوضح خطوات نشر موقع CurlyEllie على منصة Railway بشكل كامل وآمن. المشروع يعتمد على Node.js و React مع قاعدة بيانات MySQL.

## المتطلبات الأساسية

قبل البدء، تأكد من توفر الآتي:

- حساب GitHub نشط
- حساب Railway نشط
- مستودع GitHub يحتوي على كود المشروع (https://github.com/curlyellieglobal-alt/AstroPharma)
- معرفة أساسية بـ Git و GitHub

## خطوات النشر

### الخطوة 1: إنشاء مشروع جديد على Railway

1. اذهب إلى [Railway.app](https://railway.app)
2. قم بتسجيل الدخول باستخدام حسابك على GitHub
3. انقر على **"New Project"**
4. اختر **"Deploy from GitHub Repo"**

### الخطوة 2: ربط مستودع GitHub

1. اختر المستودع **"AstroPharma"** من قائمة المستودعات
2. اختر الفرع **"main"** (الفرع الرئيسي)
3. انقر على **"Deploy"**

سيبدأ Railway في بناء المشروع تلقائياً.

### الخطوة 3: إضافة قاعدة بيانات MySQL

1. في لوحة تحكم مشروعك على Railway، انقر على **"+ New"**
2. اختر **"Database"** ثم **"MySQL"**
3. سيتم إنشاء قاعدة بيانات جديدة تلقائياً

### الخطوة 4: الحصول على بيانات الاتصال بقاعدة البيانات

1. انقر على خدمة MySQL التي تم إنشاؤها
2. انتقل إلى تبويب **"Variables"** أو **"Connect"**
3. ابحث عن **"DATABASE_URL"** أو **"Connection String"**
4. انسخ هذه القيمة (عادةً تبدأ بـ `mysql://`)

### الخطوة 5: تكوين متغيرات البيئة

1. انقر على خدمة التطبيق (الخدمة الأولى التي تم إنشاؤها)
2. انتقل إلى تبويب **"Variables"**
3. أضف المتغيرات التالية:

| المتغير | القيمة | الملاحظات |
|---------|--------|---------|
| `DATABASE_URL` | `mysql://user:password@host:port/database` | انسخ من قاعدة البيانات |
| `NODE_ENV` | `production` | بيئة الإنتاج |
| `PORT` | `3000` | المنفذ الافتراضي |
| `JWT_SECRET` | `your_jwt_secret_here` | مفتاح سري قوي (استخدم قيمة عشوائية طويلة) |
| `VITE_APP_ID` | `your_app_id_here` | معرف التطبيق من Manus |
| `OAUTH_SERVER_URL` | `https://auth.manus.im` | خادم المصادقة |
| `OWNER_OPEN_ID` | `your_owner_open_id` | معرف المالك من Manus |
| `BUILT_IN_FORGE_API_URL` | `https://api.manus.im` | خادم التخزين (اختياري) |
| `BUILT_IN_FORGE_API_KEY` | `your_api_key_here` | مفتاح API (اختياري) |

**ملاحظة مهمة:** احرص على عدم مشاركة هذه المتغيرات مع أحد، خاصة `JWT_SECRET` و `BUILT_IN_FORGE_API_KEY`.

### الخطوة 6: بدء النشر

بعد إضافة جميع المتغيرات، سيبدأ Railway تلقائياً في:

1. بناء المشروع (Build)
2. نشر التطبيق (Deploy)

يمكنك مراقبة التقدم من خلال تبويب **"Deployments"**.

### الخطوة 7: تشغيل ترحيلات قاعدة البيانات

بعد نشر التطبيق بنجاح:

1. انقر على آخر نشر ناجح في تبويب **"Deployments"**
2. انقر على **"Connect"** ثم **"New Terminal"**
3. قم بتشغيل الأمر التالي:

```bash
npm run db:push
```

هذا الأمر سيقوم بإنشاء جميع الجداول المطلوبة في قاعدة البيانات.

### الخطوة 8: التحقق من النشر

1. انتظر حتى ينتهي النشر بنجاح
2. انقر على رابط التطبيق (عادةً يكون في أعلى الصفحة)
3. تحقق من أن الموقع يعمل بشكل صحيح

## استكشاف الأخطاء وإصلاحها

### مشكلة: 502 Bad Gateway

**الحل:**
- تحقق من سجلات النشر (Deployment Logs)
- تأكد من أن جميع متغيرات البيئة مضبوطة بشكل صحيح
- تأكد من أن `DATABASE_URL` صحيح تماماً
- جرب إعادة النشر (Redeploy)

### مشكلة: فشل الاتصال بقاعدة البيانات

**الحل:**
- تحقق من أن `DATABASE_URL` يحتوي على البيانات الصحيحة
- تأكد من أن قاعدة البيانات MySQL تعمل بشكل صحيح
- تحقق من أن خدمة MySQL متصلة بنفس المشروع

### مشكلة: فشل الأمر `npm run db:push`

**الحل:**
- تأكد من أنك في الطرفية الصحيحة للتطبيق (وليس قاعدة البيانات)
- تحقق من أن `drizzle-kit` مثبت في `package.json`
- جرب الأمر مرة أخرى

## معلومات إضافية

### إعادة النشر

إذا أردت إعادة نشر التطبيق بعد إجراء تغييرات:

1. قم برفع التغييرات إلى GitHub
2. سيقوم Railway تلقائياً بكشف التغييرات وإعادة النشر

### مراقبة السجلات

يمكنك مراقبة سجلات التطبيق من خلال:

1. انقر على خدمة التطبيق
2. انتقل إلى تبويب **"Logs"**
3. شاهد جميع الأحداث والأخطاء

### إدارة متغيرات البيئة

إذا احتجت إلى تغيير متغير بيئي:

1. انتقل إلى تبويب **"Variables"**
2. عدّل القيمة المطلوبة
3. سيقوم Railway تلقائياً بإعادة نشر التطبيق

## الدعم والمساعدة

إذا واجهت أي مشاكل:

1. تحقق من [توثيق Railway الرسمية](https://docs.railway.app)
2. راجع سجلات النشر والأخطاء
3. تأكد من أن جميع المتطلبات مستوفاة
